    Dicionario de dados

    Aqui está o dicionário de dados para os valores brutos do dataset:

    1. id

      • Descrição: Um ID que representa uma linha dentro do conjunto de dados.
      • Tipo de Dados: Integer.
      • Valores Permitidos: [309652396, 498764591, 504868288, ...].

    2. escolaridade

    3. Descrição: Nível de escolaridade do cliente.
    4. Tipo de Dados: String.

    5. Valores Permitidos: [nan, 'nivel medio', 'pos graducao', 'ensino superior'].

    6. renda_mensal_informal

    7. Descrição: Renda bruta mensal do cliente.
    8. Tipo de Dados: Float.

    9. Valores Permitidos: [nan, 1894.5078, 130.4699, ...].

    10. dependentes

    11. Descrição: Informa se o cliente possui dependentes.
    12. Tipo de Dados: String.

    13. Valores Permitidos: ['S', 'N'].

    14. estado_civil

    15. Descrição: O estado civil do cliente.
    16. Tipo de Dados: String.

    17. Valores Permitidos: ['solteiro', 'casado(a) com comunhao de bens', 'casado(a) com comunhao parcial de bens', 'casado(a) com separacao de bens', 'divorciado', 'separado judicialmente', 'viuvo(a)', 'outros'].

    18. idade

    19. Descrição: A idade do cliente na data do pedido de empréstimo.
    20. Tipo de Dados: Integer.

    21. Valores Permitidos: [36, 39, 26, 23, ...].

    22. conta_poupanca

    23. Descrição: Informa se a conta é poupança.
    24. Tipo de Dados: String.

    25. Valores Permitidos: ['S', 'N'].

    26. conta_salario

    27. Descrição: Informa se a conta é salário.
    28. Tipo de Dados: String.

    29. Valores Permitidos: ['S', 'N'].

    30. qtd_fonte_renda

    31. Descrição: A quantidade de fontes de renda do cliente.
    32. Tipo de Dados: Float.

    33. Valores Permitidos: [nan, 1., 2., 3.].

    34. cheque_sem_fundo

    35. Descrição: Informa se houve cheque sem fundo.
    36. Tipo de Dados: String.

    37. Valores Permitidos: ['S', 'N'].

    38. conta_conjunta

      • Descrição: Informa se a conta é conjunta.
      • Tipo de Dados: String.
      • Valores Permitidos: ['S', 'N'].

    39. valor_conta_corrente

      • Descrição: Valor na conta corrente do cliente.
      • Tipo de Dados: Float.
      • Valores Permitidos: [nan].

    40. valor_conta_poupanca

      • Descrição: Valor na conta poupança do cliente.
      • Tipo de Dados: Float.
      • Valores Permitidos: [nan].

    41. valor_emprestimo

      • Descrição: Valor do empréstimo solicitado pelo cliente.
      • Tipo de Dados: Float.
      • Valores Permitidos: [8174.32, 41775.63, 522940.27, ...].

    42. multa

      • Descrição: Valor da multa associada ao empréstimo.
      • Tipo de Dados: Float.
      • Valores Permitidos: [263.25, 1607.29, 24860.4, ...].

    43. juros

      • Descrição: Valor dos juros associados ao empréstimo.
      • Tipo de Dados: Float.
      • Valores Permitidos: [5.8449800e+03, 7.6522600e+03, 3.1931562e+05, ...].

    44. valor_emprestimo_atualizado

      • Descrição: Valor atualizado do empréstimo (principal + juros + multa).
      • Tipo de Dados: Float.
      • Valores Permitidos: [14282.55, 51035.18, 867116.29, ...].

    45. genero

      • Descrição: Gênero do cliente.
      • Tipo de Dados: String.
      • Valores Permitidos: ['feminino', 'masculino'].

    46. data

      • Descrição: Data do registro.
      • Tipo de Dados: String (formato de data).
      • Valores Permitidos: ['2020-01-01'].

    47. estado

      • Descrição: Estado onde o cliente está localizado.
      • Tipo de Dados: String.
      • Valores Permitidos: ['minas gerais', 'espirito santo', 'sao paulo', 'rio de janeiro'].

    48. pago

      • Descrição: Indica se o empréstimo foi pago ou não.
      • Tipo de Dados: String.
      • Valores Permitidos: ['sim', 'nao'].
